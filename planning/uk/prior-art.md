# Factorio Reverse Diagrams — Попередні роботи та аналоги

Каталог існуючих реалізацій, що вирішують проблеми, ідентичні або близькі до конкретних діаграм нашого плану. Для кожного оцінюємо: що робить, як побудований, що можемо запозичити, що потрібно змінити.

---

## 1. Золотий стандарт: Bartosz Ciechanowski (ciechanow.ski)

**Що це:** Довгі інженерні статті з 50–100+ вбудованими інтерактивними 3D/2D симуляціями на статтю. Теми: механічні годинники, шестерні, GPS, двигуни внутрішнього згоряння, камери, Земля та Сонце.

**Чому важливо для нас:** Найближча існуюча реалізація до того, що ми хочемо побудувати. Кожна стаття Ciechanowski структурно ідентична розділу нашого проєкту: прозові пояснення з вбудованими інтерактивними віджетами, поступове розкриття, наратив через скролінг.

**Технічна реалізація:**
- Власний WebGL рендеринг (не Three.js — рукописний `watch.js`, `gears.js` тощо)
- Без React чи фреймворків — чистий JS з власним рендер-пайплайном
- Кожна стаття — одна HTML-сторінка з вбудованими `<canvas>` елементами
- Анімації з кнопкою play/pause; глобальна пауза впливає на всі віджети
- Перетягування для обертання 3D-моделей; повзунки для зміни параметрів
- Надзвичайно відполірований візуальний дизайн — темний фон, послідовна палітра

**Що можемо запозичити:**
- Один розробник здатен виробити таку якість (Ciechanowski працює самостійно)
- WebGL підходить для складних механічних симуляцій у браузері
- Співвідношення тексту до віджетів має значення: достатньо тексту для контексту, але не стільки, щоб поховати інтерактиви
- Глобальне керування анімацією (пауза всього) — необхідне для зручності читання
- Фреймворк не потрібен, якщо масштаб однієї сторінки помірний

**Що ми б змінили:**
- Нам потрібна багатосторінкова навігація (у Ciechanowski — незалежні статті)
- Нам потрібна i18n (у нього — ні)
- Нам потрібен контент, керований даними (у нього контент закодований вручну під кожну статтю)
- Нам, ймовірно, не потрібен повний WebGL для більшості діаграм — Canvas2D та SVG достатні для 2D механік Factorio

**Релевантність:** ВСІ частини. Це наш еталон формату.

**URL:** https://ciechanow.ski/

## 2. Red Blob Games — Туторіал A* Pathfinding (redblobgames.com)

**Що це:** Інтерактивний туторіал з алгоритмів пошуку на графах (BFS, Dijkstra, A*) з використанням сіткових карт із перетягуваними початковими/кінцевими точками, анімованим розширенням фронту та візуалізацією вартості.

**Чому важливо для нас:** Безпосередньо стосується Частини II.3 (Pathfinding поїздів) та Частини VII (Генерація карти). Стиль реалізації Аміта Пателя — вбудовані інтерактивні сітки з контролями параметрів — саме те, що потрібно нашому симулятору стрічок та тепловій карті забруднення.

**Технічна реалізація:**
- Власний JS + SVG/Canvas (без фреймворку в старіших статтях; Vue.js в деяких новіших)
- Рендеринг сітки як SVG або Canvas залежно від складності
- Drag-and-drop для розміщення стін, початкової, кінцевої точок
- Візуалізація алгоритму: покрокова або безперервна анімація з контролем швидкості
- Порівняння алгоритмів пліч-о-пліч
- Реактивний потік даних: зміна одного входу запускає перерахунок і перемальовку

**Що можемо запозичити:**
- Сіткові симуляції чудово працюють з SVG для малих сіток та Canvas для великих
- Візуалізація алгоритму найефективніша, коли користувач може крокувати тік-за-тіком (паралель з нашим патерном "покрокове проходження тіків")
- Порівняння пліч-о-пліч — потужний засіб для діаграм "до/після" (fluid 1.1 vs 2.0, furnace vs foundry)
- Аміт Патель детально писав про свій підхід до реалізації (див. simblob.blogspot.com) — безцінний технічний довідник
- Патерн Observer для зв'язування кількох віджетів на одній сторінці

**Що ми б адаптували:**
- Pathfinding поїздів: замінити квадратну сітку на граф рейок; показати значення штрафів на ребрах; анімувати вибір поїздом найдешевшого шляху
- Дифузія забруднення: замінити фронт pathfinding на хвильовий фронт дифузії; кожна клітинка має значення забруднення замість бінарного відвідано/не відвідано
- Механіка стрічок: замінити сітку на лінійний масив слотів; анімувати рух предметів

**Релевантність:** Частини II.3, V.1, VII.1

**URLs:**
- https://www.redblobgames.com/pathfinding/a-star/introduction.html
- https://www.redblobgames.com/grids/hexagons/
- https://simblob.blogspot.com/2018/02/how-i-implement-my-interactive-diagrams.html (нотатки про реалізацію)

## 3. Setosa — "Markov Chains Explained Visually" (setosa.io/ev/markov-chains)

**Що це:** Інтерактивне explorable explanation ланцюгів Маркова з редагованою матрицею переходів, анімованою діаграмою станів та візуалізацією потоку ймовірностей.

**Чому важливо для нас:** Безпосередньо стосується Частини VIII.6 (Система якості). Гринд якості у Factorio Є ланцюгом Маркова — Normal → Uncommon → Rare → Epic → Legendary, з ймовірностями переходу, визначеними модулями якості.

**Технічна реалізація:**
- D3.js для рендерингу діаграми станів
- Текстові поля для редагування матриці переходів
- Анімовані токени, що рухаються між станами
- Обчислення стаціонарного розподілу в реальному часі
- Компактний, зосереджений — одна концепція, одна сторінка

**Що можемо запозичити:**
- Редаговані матриці переходів інтуїтивні для користувачів, що розуміють математику
- Анімований потік ймовірностей (частинки, що течуть вздовж ребер пропорційно ймовірності переходу) робить абстрактну математику відчутною
- Обчислення стаціонарного стану поряд з діаграмою допомагає зрозуміти довгострокову поведінку

**Що ми б адаптували:**
- Стани стають рівнями якості (від Normal до Legendary)
- Ймовірності переходу визначаються конфігурацією модулів якості (обирається користувачем)
- Додати обчислення "очікуваної кількості ітерацій до цільової якості"
- Додати вартість ресурсів за ітерацію (інгредієнти × очікувані ітерації = загальна вартість)
- Додати цикл рисайклера: не та якість → переробка → повторне виготовлення, як додаткові ребра переходів

**Існуючі Factorio-специфічні моделі Маркова:** Декілька членів спільноти опублікували калькулятори гринду якості (на базі таблиць), але жоден — як інтерактивний візуальний ланцюг Маркова. Це було б новинкою.

**Релевантність:** Частина VIII.6

**URL:** https://setosa.io/ev/markov-chains/

## 4. Nicky Case — "Loopy" та Explorable Explanations (ncase.me)

**Що це:** Інтерактивні інструменти для системного мислення. "Loopy" дозволяє малювати діаграми причинно-наслідкових петель та симулювати їх. Інші explorables ("Parable of the Polygons", "Evolution of Trust") використовують ігрові взаємодії для навчання системних концепцій.

**Чому важливо для нас:** Безпосередньо стосується Частини X (Система систем). Наша головна діаграма зворотних зв'язків — що показує, як забруднення, бій, виробництво, дослідження та енергія взаємопов'язані — саме той тип причинно-наслідкової петлі, який Loopy візуалізує.

**Технічна реалізація:**
- "Loopy": власний JS canvas-додаток; користувачі малюють вузли та ребра; симуляція запускає потік вздовж ребер; позитивний/негативний зворотний зв'язок маркований
- Інші explorables: власний HTML/JS під кожен проєкт; ігрові взаємодії; багато анімацій та агентності користувача
- Відкритий код (public domain)

**Що можемо запозичити:**
- Діаграми причинно-наслідкових петель мають бути ІГРОВИМИ — не просто статичними графами вузлів та зв'язків
- "Що станеться, якщо видалити цей зв'язок?" — ключова взаємодія
- Позитивні та негативні петлі зворотного зв'язку мають візуально відрізнятися (підсилюючі vs. балансуючі)
- Проста симуляція потоку через систему робить абстрактний зворотний зв'язок відчутним

**Що ми б адаптували:**
- Попередньо побудована діаграма системи систем Factorio (не намальована користувачем)
- Кожен вузол посилається на сторінку розділу відповідної Частини
- Перемикач: "видалити забруднення" → спостерігати, як руйнується бойова петля; "видалити живлення" → спостерігати, як колапсує виробнича петля
- Анімовані частинки, що течуть через ребра, показуючи причинно-наслідкові ланцюги

**Релевантність:** Частина X

**URLs:**
- https://ncase.me/loopy/
- https://ncase.me/trust/
- https://blog.ncase.me/explorable-explanations/ (принципи дизайну)

## 5. Інструменти спільноти Factorio (factoriolab, Kirk McDonald, Foreman2)

**Що це:** Калькулятори виробничих пропорцій та генератори блок-схем для Factorio.

**Чому важливо для нас:** Ці інструменти вирішують Частину III.4 (DAG рецептів) — вони вже парсять дані рецептів та візуалізують виробничі ланцюги. Нам не потрібно перебудовувати це з нуля.

**Реалізації:**

### FactorioLab (factoriolab.github.io)
- Angular додаток; повна база рецептів для vanilla + SA + моди
- Діаграма Санкі на виході, що показує потік ресурсів
- Альтернатива у вигляді блок-лінійної діаграми
- Конфігурація модулів та маяків
- Підтримка якості (Space Age)
- Відкритий код: github.com/factoriolab

### Kirk McDonald Calculator (kirkmcdonald.github.io)
- Чистий JS; діаграма Санкі з використанням D3-sankey
- Старіший, але добре зарекомендований
- Простіший UI, зосереджений на обчисленні пропорцій

### Foreman2 (github.com/DanielKote/Foreman2)
- Десктопний додаток (Windows); вузлова блок-схема
- Drag-and-drop побудова графу рецептів
- Не веб-базований

**Що можемо запозичити:**
- Парсинг даних рецептів вирішений — FactorioLab вже обробляє `data.raw` включно з Space Age
- Діаграми Санкі для виробничого потоку добре зрозумілі спільноті Factorio
- Обчислення якості вже реалізовані в FactorioLab

**Що ми б зробили інакше:**
- Наш DAG рецептів — не калькулятор, а ПОЯСНЕННЯ структури
- Ми показуємо повну топологію графу (що з чим з'єднано), а не лише пропорції для цільового предмета
- Ми додаємо прозовий контекст: ЧОМУ вузол зелених мікросхем так сильно зв'язаний? ЩО це означає для дизайну фабрики?
- Ми дозволяємо дослідження: клік на будь-який вузол показує його підграф, залежності та залежних
- Ми потенційно повторно використовуємо рівень даних FactorioLab або посилаємось на нього для детальних обчислень

**Релевантність:** Частини III.4, VIII.14

**URLs:**
- https://factoriolab.github.io
- https://kirkmcdonald.github.io
- https://github.com/DanielKote/Foreman2

## 6. CelLab — Клітинні автомати в браузері (fourmilab.ch/cellab)

**Що це:** Браузерний симулятор клітинних автоматів. Користувачі можуть визначати правила, створювати початкові патерни та спостерігати за їх еволюцією. Включає попередньо визначені правила для теплового потоку, дифузії, хімічних реакцій.

**Чому важливо для нас:** Безпосередньо стосується Частини V.1 (Дифузія забруднення). Система забруднення Factorio Є клітинним автоматом — кожен чанк має значення забруднення, і кожен тік забруднення дифундує до сусідніх чанків з поглинанням.

**Технічна реалізація:**
- JavaScript + HTML5 Canvas (раніше Java-аплет)
- Сітка клітинок з кольоровим кодуванням станів
- Визначення правил користувачем через JavaScript
- Контролі анімації: запуск, крок, швидкість
- Попередньо визначені правила для дифузії, теплового потоку, реакції-дифузії

**Що можемо запозичити:**
- Дифузія на сітці добре зрозуміла та ефективна для симуляції в браузері
- Кольорове відображення (теплова карта) — природна візуалізація
- Користувач має мати змогу розміщувати "джерела" (фабрики) та "стоки" (дерева) і спостерігати еволюцію забруднення
- Покроковий режим необхідний для розуміння правила дифузії

**Що ми б адаптували:**
- Фіксована сітка з масштабом чанків Factorio (не довільні правила КА)
- Джерела забруднення = фабричні сутності з відомими значеннями забруднення/хвилину
- Поглинання = залежне від типу тайлу (тайли дерев поглинають більше, ніж пустелі)
- Показувати числові значення по чанках, а не лише кольори
- Накладення: коли забруднення досягає спавнера, показати тригер атаки бітерів

**Також релевантне:** Туторіал дифузії Biological Modeling (biologicalmodeling.org) — Python/Jupyter реалізація дифузійного КА з лапласівським ядром згортки. Математика ідентична моделі забруднення Factorio.

**Релевантність:** Частина V.1

**URL:** https://www.fourmilab.ch/cellab/

## 7. Factsim — Симулятор ланцюгової мережі Factorio (github.com/Factsimguru/Factsim)

**Що це:** Python-інструмент, що імпортує рядки блюпринтів Factorio, витягує з'єднання ланцюгової мережі та симулює логіку комбінаторів тік-за-тіком з візуальною діаграмою.

**Чому важливо для нас:** Безпосередньо стосується Частин VI.2–6.3 (Комбінатори, Схемні патерни). Це єдиний існуючий інструмент, що симулює ланцюгову мережу Factorio поза грою.

**Технічна реалізація:**
- Python 3 з GUI на tkinter
- Парсинг JSON блюпринту для витягу сутностей та дротових з'єднань
- Рендеринг діаграми ланцюга з іконками сутностей та дротовими з'єднаннями
- Покрокова симуляція: крок вперед/назад, інспекція значень сигналів по сутності по тіку
- Обробка арифметичних, вирішувальних, константних комбінаторів

**Що можемо запозичити:**
- Парсинг рядків блюпринтів простий (base64 → JSON → список сутностей)
- Тік-точна симуляція логіки комбінаторів здійсненна та не є обчислювально дорогою
- Покрокова інспекція (переглянути всі сигнали на тіку N) — ключова взаємодія
- 1-тікова затримка на комбінатор правильно змодельована

**Що ми б адаптували:**
- Портувати у веб (JavaScript/TypeScript) замість Python/tkinter
- Візуальний дизайн: використовувати іконки у стилі Factorio, кольори дротів, іконки сигналів
- Додати selector combinator (2.0) та display panel
- Додати бібліотеку попередньо побудованих патернів (SR-засувка, годинник, диспетчер поїздів)
- Зробити вбудовуваним як віджет всередині нашої сторінки розділу
- Дозволити користувачам модифікувати схеми (додавати/видаляти комбінатори, змінювати налаштування) без імпорту блюпринтів

**Релевантність:** Частини VI.2, VI.3

**URL:** https://github.com/Factsimguru/Factsim

## 8. Game Mechanic Explorer (gamemechanicexplorer.com)

**Що це:** Колекція інтерактивних JavaScript-демо для ігрових механік (рух, зіткнення, pathfinding, частинки тощо), кожна з прогресивною складністю від базової до просунутої.

**Чому важливо для нас:** Педагогічна структура — почати просто, поступово додавати складність — саме те, що нам потрібно для кожної Частини. Напр., механіка стрічок: почати з однієї лінії, додати предмети, додати дві лінії, додати бокове завантаження, додати сплітер...

**Технічна реалізація:**

## 9. Machinations Framework (machinations.io)

**Що можемо запозичити:**
- Діаграми ресурсних потоків — потужна абстракція для ігор-фабрик
- Словник пул/джерело/стік/конвертер безпосередньо відповідає сутностям Factorio (скриня=пул, бур=джерело, споживач=стік, збирач=конвертер)
- Симуляція ресурсного потоку робить вузькі місця видимими негайно

**Що ми б адаптували:**

## 12. Factorio-SAT — Оптимальні макети стрічок з SAT-солверами (github.com/R-O-C-K-E-T/Factorio-SAT)

**Що це:** Python-інструмент, що використовує SAT (Boolean satisfiability) солвери для знаходження доказово оптимальних макетів балансерів стрічок. Кожен тайл моделюється як булеві змінні (напрямок, підземний стан, сторона сплітера); обмеження забезпечують валідність стрічок; солвер знаходить мінімальні макети.

**Технічна реалізація:**
- Python + SAT солвер (Cadical/Glucose)
- Тайли закодовані як булеві змінні: напрямок (4), підземний вхід/вихід, ліва/права сторона сплітера, колір
- Обмеження кодують: зв'язність стрічок, парування сплітерів, правила підземного тунелювання, збереження потоку
- Виходи: рядки блюпринтів, відрендерені зображення, анімовані GIF потоку предметів

**Що можемо запозичити:**
- Кодування обмежень Є формальною специфікацією правил стрічок — саме те, що наші діаграми мають пояснити неформально
- Анімований рендерер блюпринтів вже візуалізує потік предметів на стрічках
- Графи мереж сплітерів (згенеровані через Graphviz) подібні до того, що ми б малювали для Частини II.1

**Що ми б адаптували:**
- Використати описи обмежень як довідку для наших прозових пояснень ("тайл стрічки має напрямок D, приймає вхід від тайлів, спрямованих на D...")
- Анімаційний рендерер може бути адаптований для нашого віджету симулятора стрічок
- Ми б не використовували SAT-розв'язання саме по собі — але формальна модель інформує контент наших діаграм

**Також пов'язане:** `alegnani/verifactory` — верифікатор блюпринтів Factorio з використанням Z3 SMT солвера. Парсить рядки блюпринтів, конвертує в логічні формули, доводить властивості балансування стрічок. Ще одна формальна модель механіки стрічок.

**Релевантність:** Частина II.1 (формальна модель стрічок), Частина III.4 (мислення на основі обмежень)

**URLs:**

## 18. Реєстр клонів ігор з відкритим кодом — Список клонів Factorio

**Що це:** Курований список клонів та реімплементацій комерційних ігор з відкритим кодом, включаючи Factorio.
